# Anti-Patterns Verdict

**Does this look AI-generated? No.** Product-register test (would a Strong/Hevy-fluent user trust it): pass. One accent doing real state work, mono reserved for data, undo-toast over modal, modals only for destructive confirms, hand-tuned plate math. All shared bans clean: no side-stripes, no gradient text, no decorative glass, no hero-metric template, no identical card grids.

**Deterministic scan** (27 patterns): 4 warnings, no errors. 3× cramped-padding, all the same real issue: the `.sheet` dialogs' scroll body ends flush at the bottom with no inset. 1× all-caps-body (32 chars): borderline false positive; every uppercase use in the app is a 9-11px section label, not prose.

**Where they agree**: both assessments found nothing in the slop catalogue. The detector caught the dialog bottom-padding issue the LLM review missed; the LLM review caught everything else.

**Overlay note**: the installed impeccable CLI lacks the `live` command, so no in-browser overlay this run; CLI findings stand alone.

# Overall Impression

A genuinely well-crafted product with one screen carrying nearly all the debt: the strength card on the day view. The system thinking (undo architecture, rest timer with filler, restrained blueprint aesthetic) is better than most paid apps in this category. But the highest-traffic surface, the set row mid-workout, is the most cluttered and its primary action the least visible; a fresh program greets the user with `planned 0kg × –` machine output; and a finished workout ends by fading the user's work to 55% opacity with no closing moment.

# What's Working

1. **The undo architecture** (`toastUndo`): delete session, reset day, even shrinking a set count snapshots logged data first. Linear-grade respect for user work.
2. **The rest timer + filler**: auto-starts from the actual prescription, screen-reader announced, suggests a swap-able low-fatigue accessory. Best-in-category thinking.
3. **Visual restraint**: one accent, mono-for-data, hairline forms, no decoration that does not encode state. The week view (name / focus / 0-of-6 / chevron) is exactly what a glanceable index should be.

# Priority Issues

**[P0] `planned 0kg × –` renders on every set row of every freshly built program**
Why: cryptic machine output on the app's highest-traffic surface; makes planned-vs-actual look broken before it has data. Fix: suppress the caption when `target_weight` is 0/falsy and `target_reps == null` (one condition in `setRowsInner`). Suggested command: /impeccable polish

**[P1] Strength set row exposes 5 controls; the primary action is the least visible**
23 visible tappables on one card (weight -/input/+, then per row: weight chip, rep-, rep+, RPE, done circle). Mid-set, the one control that matters (done) is rightmost and transparent until tapped. Fix: make the done circle visually dominant; demote RPE + per-set weight override (reveal on the just-completed row, or behind one "…"). Suggested command: /impeccable distill

**[P1] No end-of-session moment**
Last set of the last exercise: the card dims and a 10px "DONE" appears. Peak-end rule inverted; the session closes by subtraction. Fix: one-shot toast/sheet on `done === total` with session totals ("Upper A done: 22 sets, est-1RM up on 2 lifts") + "Back to week". Suggested command: /impeccable delight

**[P2] Scoring vocabulary unexplained + stale intro copy**
Visible "RPE" pill per row (its own aria-label already says "Effort"); `target 3 RIR · RPE 7`; "est. 1RM"; intro bullet still says "open Form notes" though the toggle was renamed "Details" (v53 leftover). Fix: s/Form notes/Details/ in the intro; label the pill "Effort"; one-time hint on first RPE tap. Suggested command: /impeccable clarify

**[P2] First-run weight = 0 with no prompt**
Warm-up pyramid, plates line, progression, est-1RM all stay dormant until a weight is entered, and nothing tells the user that. Fix: style the weight field as the card's call-to-action until weight > 0 (placeholder "set your weight"). Suggested command: /impeccable onboard

**[P3] Sheet dialogs end flush at the bottom (detector)** + unlabeled topbar glyphs. Suggested command: /impeccable polish

# Persona Red Flags

**Casey (distracted mobile user, one-handed mid-set — the primary persona)**: 23 tap targets per strength card; rep- sits adjacent to the weight chip (mis-tap risk with sweaty thumbs); primary "build" action is a top-right icon outside the thumb zone. Passes: state fully persists across interruptions; rest timer and its Skip are bottom-anchored in the thumb zone.

**Jordan (first-timer, non-gym-expert — the stated project goal)**: intro instructs opening "Form notes" which no longer exists; RPE/RIR/1RM never explained; `planned 0kg × –` reads as broken; icon-only topbar with the empty state calling it "the program button up top". Passes: skill gate now prevents the handstand-push-up ambush; inline Setup/Key/Fault cues under each name; plain confirm copy before destructive actions.

**Alex (power user — the owner)**: efficiency path is strong (whole-card tap target, seeded "Try again", Just-today free text, per-set overrides). Red flag: no session summary means progression decisions ("ready to add weight" on which lifts?) must be gathered card by card after finishing.
